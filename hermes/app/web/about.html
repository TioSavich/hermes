What this is

Hermes is a local console for teachers of elementary mathematics. It runs on your machine and reads a knowledge base that encodes children's arithmetic strategies as runnable machines, misconceptions documented in research literature, content standards, and lesson context from the Illustrative Mathematics curriculum. When you ask it something, the answer comes from that knowledge base, with its sources attached.

The surfaces

Console
Type a computation and the calculator works it the way a child using a named strategy would, drawing each action on a number line. The encyclopedia holds the strategies, the documented misconceptions (where an entry carries a runnable rule, the console runs it and produces the exact wrong answer it predicts), the standards, and the grounding metaphors together with the places where each metaphor stops working. The lesson lab gathers what the knowledge base holds about a particular IM lesson.
Discussions
Paste a classroom transcript and the console checks the mathematical claims it can recognize, deterministically: each claim comes back holds, refuted, or not covered, with the worked check attached. The fuller two-pass report asks a language model first, so it needs a key.
Visualizers
Number line, fraction bars, area model, base-ten blocks, set & grouping, and balance scale. The same drawing grammar the calculator uses, available directly.
Monitoring charts
For an IM lesson: its standards, the strategies to anticipate, and the misconceptions research has documented near its content, arranged the way a monitoring sheet arranges them.

What it does not claim

The learner this console reasons about is a formal model built from research literature. It is not your student. When a panel places a piece of student work near a strategy or a misconception, it is offering a hypothesis for you to test in conversation with the child, along with the citations and the worked trace that motivated it.

A misconception match is a reason to listen more closely. The console has no access to why a particular child wrote what they wrote, and it does not diagnose children.
Some questions come back not covered. That answer is deliberate. Where its formal ground runs out, the console stops rather than improvising, and it tells you where the boundary is.

What stays on your machine

The calculator, encyclopedia, lesson lab, claim checks, and every visualizer run entirely on your machine. No account, no network, no key.

Four surfaces ask a language model (IU's REALLMS service) for prose: the Ask-Hermes chat, the PML reading of a passage, transcript reports, and grading suggestions. Each of these says so and stops when no key is configured — the chat still answers from the knowledge base without one. The discussions page states which routes analyze text locally and which send an uploaded file to the model service.

Keys, rosters, and anything you upload live in a runtime folder that version control never touches.

Hermes is the console of the UMEDCTA formalization repository. Panels cite their sources, and the numbers they report trace to predicates you can run yourself in SWI-Prolog. Launch: bash hermes/app/launch.sh, then open http://127.0.0.1:8765/.
