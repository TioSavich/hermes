# Task 117: Embedding-form interface beyond misconceptions

Status: PARTIAL

## Result

`scripts/research/misconception_embedding.py` is now one embedding engine with
thin domain definitions for misconceptions, lessons, registered strategy
automata, and dispatch operations. The existing misconception command remains
the default. Each row now has a stable identifier and exact composed text
before it reaches the embedding API.

The REALLMS embedding endpoint was unreachable in this environment. Its host
lookup failed with:

```text
embedding API call failed after 1 attempts: network: <urlopen error [Errno 8] nodename nor servname provided, or not known>
```

Following the brief's partial-success rule, the run stopped at the API seam and
wrote the exact row payloads for all three new domains. It did not create
vector archives or index sidecars for those domains.

## Existing index and embedding source

The existing misconception index establishes the shared format:

| Property | Verified value |
| --- | --- |
| Rows | 2,056 |
| Embedding model | `Qwen3-Embedding-8B` |
| Dimensions | 4,096 |
| Vector storage | float32 NumPy v1 bytes named `vectors.npy` inside a compressed `.npz` archive |
| Metadata storage | JSON sidecar with model, count, dimensions, vector filename, and row metadata |
| Endpoint | REALLMS `/embeddings`, derived from the configured `/chat/completions` URL |
| Rebuild boundary | Network-bound and local; credentials load from the repository `.env` without being printed |

The generalized writer retains this float32 `.npz` plus JSON-sidecar mechanism.
It adds a source hash so an unchanged model and unchanged row payload skip the
network call and leave the artifacts untouched. The `.npz` writer also uses a
fixed archive timestamp.

## New domain rows

| Domain | Count | Stable identifier | Text source |
| --- | ---: | --- | --- |
| Lessons | 1,308 | `lesson_id` | Published lesson coverage joined to shipped titles, standards, registered automaton contracts or lesson strategy descriptors, and compiled task stems |
| Strategies | 172 | `signature` | `action_automaton_signature/4`, input and output schemas, `action_automaton_vocabulary/3` state/action labels, and a contract description |
| Registry operations | 196 | `op` | The capability registry's module, role, inputs, typed parameters, description, status, routes, pages, and lazy-load fields |

The registry count is the live dispatch surface: 193 `routed_paged` operations
plus 3 `unrouted` operations. Lazy-reachable and orphan-module capability rows
are not dispatch operations and are excluded from this index.

No published lesson row was excluded. All 1,308 published IDs resolve in the
field-context cache. For 227 cache rows without lesson metadata, the builder
uses the shipped IM lesson anchors for title and standard text. The resulting
lesson payload has a nonempty title and at least one standard for every row.
The source material has no attached strategy for 318 lessons and no recovered
task stem for 432 lessons; those fields remain empty on those rows. Of 2,153
lesson strategy records, 1,758 resolve to registered automaton contracts and
395 remain source-labelled lesson strategy descriptors.
