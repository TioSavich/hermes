## Source and format diagnosis

The live `scripts/research/extract_lesson_context.py` is the task-36
extractor. Its K-5 path requires an `_Anchor ID`, a fenced `## Full Teacher
Guide (raw extract)` region, and fixed-width text from a two-column PDF. It
recognizes major left-column headings such as `Warm-up`, `Activity N`,
`Lesson Synthesis`, and `Cool-down`. The `Student Task Statement` and
`Activity Synthesis` labels can share a physical line with a second column.
Page furniture and unsafe cross-page fragments are rejected.

The published grade 6-8 `explicit_lesson_text_source/2` clauses point to
`curriculum/scope_and_sequence/grade6.md`, `grade7.md`, and `grade8.md`.
Those files are organized by unit. Each unit has a `## Unit N` heading, a
unit-level anchor and narrative, then a `### Lessons` list. They contain
lesson titles and sequencing but no activity statements. They therefore
cannot supply the verbatim prompt layer.

The complete teacher-guide source is the local Docling corpus under
`hermes/app/runtime/experiments/gemma4_tutor/docling/full-output/TeacherLessonGuides`.
It contains one directory per published lesson:

The grade 6-8 documents differ structurally from the K-5 prepared extracts:

- The lesson ID is encoded in the directory name, such as
  `Grade7-3-7-Lesson-teacher-guide-`; there is no lesson anchor in
  `document.md`.
- The document is linear Markdown. `## Student Task Statement`,
  `## Activity Synthesis`, and `## Lesson Synthesis` are section headings
  rather than fixed-width columns. Some student-task markers are list items
  because of the source layout.
- An activity title and `## Activity Narrative` precede each activity.
  Warm-ups may appear as a plain `Warm-up` line below the title. Cool-down
  task statements occur after `## Lesson Synthesis` and before the lesson
  summary.
- Lesson summaries, glossaries, and practice problems are appended to the
  instructional sections. They require an explicit cutoff so practice
  problems are not compiled as lesson prompts.
- `picture_descriptions.md` identifies prior model-generated image
  annotations. The same annotations are embedded in `document.md`; they are
  not curriculum-authored text.

## Extractor change

The K-5 parsing functions were left unchanged. A separate grade 6-8 path now:

1. Maps each grade 6-8 guide directory to its `IM-GN-UN-LN` lesson code.
2. Restricts extraction to the instructional body, beginning with the first
   `## Activity Narrative` and ending before the lesson summary, glossary, or
   practice problems.
3. Copies only labelled student-task and synthesis regions. Text remains in
   source order, and each item records the line of its first retained source
   line.
4. Matches `picture_descriptions.md` annotations back to `document.md` and
   removes those annotations and image links. This keeps prior generated
   descriptions out of the verbatim artifact.
5. Emits `compiled_lesson_context_absent/3` when a middle-school guide lacks
   a recoverable activity statement. The predicate is present and empty in
   this regeneration because no lesson is absent.

Thirty-eight task sections contained only an image link or a prior
model-generated description. Those sections were rejected and counted as
`middle_school_empty_or_model_only_task_statement`. Every affected lesson
also had at least one curriculum-authored task statement, so lesson coverage
was not reduced.
