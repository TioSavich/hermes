# Task 119 report — observed-table start-root repair

## Root cause

The defect is exactly the naming seam the brief hypothesized, and it lives in
one function: `render_observed_transitions` in
`scripts/research/build_transition_tables.py`.

When a bounded live observation's action sequence matches the signature's
static trace, the observed rows reuse `table.states` (the state atoms already
computed for the static trace, which correctly begin at whatever the
signature's `automaton_tuple` declares as `start(...)`). When the observation
diverges — different length or different labels, as happens for three
signatures whose live run took a branch the static extractor did not capture
(documented in `docs/research/2026-07-23-task-111-report.md` as extraction
defects, not automaton defects) — the builder reconstructs a fresh chain of
`q_observed_N` states. That reconstruction hardcoded its endpoints as the
literal atoms `"q_start"` and `"q_accept"`:

```python
states = (table.states if observation.actions == table.actions else
          tuple(["q_start", *(f"q_observed_{n}" for n in range(1, len(observation.actions))), "q_accept"]))
```

`"q_start"` is the correct root only for signatures whose static trace comes
from `table_from_trace` (a bare action-pair `Trace = [...]` list, which always
synthesizes states beginning at `q_start`). The three affected signatures —
`fraction/benchmark_fraction_comparison`, `fraction/common_unit_fraction_comparison`,
`fraction/gap_thinking_fraction_comparison` — are instead built by
`table_from_history` from a comparison automaton's `hist(q_init, ...)` trace
(`knowledge/strategies/math/smr_frac_benchmark_compare.pl`,
`smr_frac_common_unit_compare.pl`), whose `automaton_tuple` declares
`start(q_init)`. For exactly these three, and only when the observed action
sequence diverged from the static one, the hardcoded literal produced a
`q_start` root disconnected from the declared `q_init` start — a dangling
first transition plus an orphaned six- or eight-state chain, invisible to
`hermes/strategy_recognizer.pl` (which reads facts in file order, not by
walking the graph from `start(...)`) but fatal to any consumer, such as Big
Red, that roots the automaton at its declared start before minimizing.

The fix replaces both hardcoded literals with the table's own declared
boundary atoms, `table.states[0]` and `table.states[-1]`, so the
reconstructed chain always roots and terminates at the atoms the signature's
own `automaton_tuple` already declares — for signatures built from
`table_from_trace` this is `table.states[0] == "q_start"` unchanged; for the
three comparison signatures it is `"q_init"`.

## New check

`scripts/checks/transition_tables.py` (new) has two parts:

1. **Rootedness** — parses every generated `automaton_tuple/6` and every
   `automaton_transition/6` fact carrying `provenance(observed(Source))`,
   groups the observed rows by `(operation, kind, source)`, and confirms
   every state touched by that group is reachable (BFS over the `Before →
   After` edges) from the signature's own declared `start(...)` atom. Run
   against the pre-fix snapshot, it reproduces the exact six-row failure this
   task fixes (`FAIL: fraction.pl: fraction/benchmark_fraction_comparison
   observed(contract_example) rows unreachable from declared start 'q_init':
   [...]`, and the same for the other two signatures and both sources — see
   Verification). Run against the regenerated tree it passes cleanly across
   all 131 observed signature/source groups.
2. **Determinism** — runs `build_transition_tables.py --check` as a
   subprocess and requires exit 0, the builder's own fast in-memory
   rebuild-and-compare, covering every family (not only fraction) as the
   full form of the two-run byte comparison.

Runtime is ~2.3 seconds (dominated by the builder's bounded probes), in line
with the other Tier-1 Python checks in `scripts/checks/`.
