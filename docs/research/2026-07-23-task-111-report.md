# Task 111 report — observed tables at scale

## Completed work

`scripts/research/build_transition_tables.py` now parses every checked input
contract, mirrors `hermes_encyclopedia:trace_inputs/3`, and runs the registry
under a two-second per-probe limit. It emits `provenance(observed(contract_example))`
facts for every successful checked example and, where the input can be changed
without changing its shape, a second bounded
`provenance(observed(derived_template))` path. A failed or timed-out derived
probe produces no fact and is reported rather than inferred.

The generated table files retain their static-source facts. Observed paths use
the static states when their labels agree; a divergent observed path receives
its own `q_observed_*` states so no static transition is recast as live
evidence.

## `co_denominator_make_base_transfer` diagnosis (report only)

The earlier timeout was not reproducible at the Prolog seam. With the same
denominator shape required by the automaton,
`fraction_pair(47,10,28,10), unit(whole)`, a two-second bounded call returned
in 0.001242 CPU seconds after 7,644 inferences. The encyclopedia seam returned
`ok:true` with the six expected labels.

A 1,000-run SWI profile took 0.656 seconds. The hot predicate was
`grounded_arithmetic:config_cost_for_action/2` (92,000 calls; 65.2% exclusive
time), reached through `sar_add_rmb:run_rmb/4` and the FSM cost accounting.
There is no current timeout to fix. If the timeout returns under a materially
larger input, the fenced follow-up should profile that exact input and reduce
the repeated cost-accounting work in `grounded_arithmetic:config_cost_for_action/2`
(for example, memoized or batched accounting that preserves the existing trace
and outcome). That change belongs outside this task's file fence.
