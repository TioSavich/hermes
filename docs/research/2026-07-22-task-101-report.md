# Task 101 report — lesson-specific notation charts

## Scope and result

Implemented the notation-chart data-path repair inside the file fence. No Git
operations were run. The live worker now selects a chart equation only from a
registered productive task for the requested lesson and the operation encoded
for that lesson. An IM lesson without such material returns one explicit
sentence rather than a chart borrowed from another lesson.

## Verification of the complaint

The pre-change generated payloads for five scattered lessons were pulled from
`hermes/app/web/generated/notation_lesson_charts/<code>/chart.json`.

| Lesson | Before operation and equation | Before deformation set |
| --- | --- | --- |
| `IM-GK-U7-L10` | addition; `3 + 2 = 5` | operational-equals chain, mirror-written numeral |
| `IM-G1-U2-L1` | addition; `3 + 2 = 5` | operational-equals chain, mirror-written numeral |
| `IM-G3-U3-L2` | addition; `8 + 4 = 12` | operational-equals chain, digit transposition, place-value writing error, carry mark |
| `IM-G3-U1-L13` | multiplication; `3 * 2 = 6` | operational-equals chain, mirror-written numeral |
| `IM-G4-U6-L5` | addition; `8 + 4 = 12` | operational-equals chain, digit transposition, place-value writing error, carry mark |

This confirms the complaint. The mechanism was the former
`representative_equation/3` default table in
`curriculum/im/lesson_notation_chart.pl:169-180` (pre-change): it chose one of
five fixed equations from operation and a multi-digit heuristic. It did not
read the lesson's registered task instances. It also allowed an attached
addition strategy to hide a lesson's registered subtraction task.

## Repair

- `curriculum/im/lesson_notation_chart.pl:91-175` imports the compiled task
  instances and requires both an encoded lesson operation and a matching
  `productive-add`, `productive-subtract`, or `productive-multiply` fact.
  The chart equation is computed from that task's operands.
- `curriculum/im/lesson_notation_chart.pl:359-370` returns the task-77-law
  payload, `No lesson-specific notation material is registered for this
  lesson.`, for any known IM lesson without a matching task-backed notation
  chart.
- `hermes_worker.pl:1183-1197` preserves that no-material payload as a normal
  response and reserves an error for an unknown lesson code.
- `hermes/web/visualizations.html:211-227` describes the lesson-specific
  behavior and removes the stale claim that the task-instance cache is unused.

## Same-five post-change diff

| Lesson | After operation and equation | Difference |
| --- | --- | --- |
| `IM-GK-U7-L10` | addition; `2 + 3 = 5` | Uses its first registered addition task instead of the shared `3 + 2 = 5`. |
| `IM-G1-U2-L1` | addition; `5 + 2 = 7` | Uses its registered books task instead of the shared `3 + 2 = 5`. |
| `IM-G3-U3-L2` | subtraction; `674 - 327 = 347` | Uses its registered subtraction task; the old operation-priority collapse had served an addition chart. |
| `IM-G3-U1-L13` | multiplication; `4 * 5 = 20` | Uses its first registered multiplication task instead of the shared `3 * 2 = 6`. |
| `IM-G4-U6-L5` | multiplication; `8 * 27 = 216` | Uses its registered multiplication task instead of the shared addition chart. |

The resulting deformation sets also now follow those lesson-specific
equations. For example, the subtraction and multi-digit multiplication charts
surface the applicable transposition and place-value-writing cases; the
addition-only carry case no longer appears merely because a default `8 + 4`
was chosen.

The no-material path was also checked with `IM-G6-U2-L11`: it returns exactly
`No lesson-specific notation material is registered for this lesson.` even
though the lesson has registered arithmetic tasks, because its encoded lesson
operation is `ratio`, not one of this notation chart's supported operations.
