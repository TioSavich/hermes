## Alignment behavior

The recognizer reads the executed transition facts in their recorded order.
This matters for `fraction/add_numerator_denominator_comparison`, whose witness
contains a self-loop with two distinct actions, and for the three fraction
comparison traces whose observed transitions differ from the static
extraction.

Partial text advances only through the matched prefix. Later matched actions
are retained as incompatible transitions when the required frontier has not
been reached. An omitted action remains missing evidence. The injected-error
fixtures therefore return a partial candidate rather than a clean run.

The current observed tables contain no witnessed commuting diamond or other
explicit permission to exchange adjacent actions. The round-trip check does
not create reordered clean traces without that evidence.
