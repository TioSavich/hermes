## Implementation

The review-pending articulation contract now records one of five candidate
kinds:

- `transformation`
- `relational_judgment`
- `precondition`
- `representation_constraint`
- `abstention`

Each generated candidate declares a typed probe and a contrast probe. Results
use a constructor associated with the declared kind. An abstention requires one
explicit `abstains(Reason)` result and a contrasting `proceeds(Result)` result.
Plain failure and echoed input do not reproduce documented behavior.

The bundle derives a required role floor from the documented account. A
fraction-of-a-number account requires `fraction`, `target_quantity`, and
`activity_context`. The registration, both probes, and the rule head must bind
the same roles.

Value results reject nested arithmetic operators. A candidate may retain an
arithmetic expression only when it declares `symbolic_expression` mode and
returns `constructed_expression(Expression)`.

Support markers are restricted to `SOURCE-ARTICULATION` and
`EDUCATED-GUESS`. Missing or unknown markers are recorded as `UNLABELLED`.

The typed gate records `loads_and_terminates` separately from
`reproduces_documented_behavior`. The second field becomes true only when both
role-complete typed probes terminate, return valid ground constructors, and
match their declared expected results.

One local retry is available for punctuation-only syntax damage, such as a
missing clause terminator. The retry reuses the same generated draft. Semantic
failures receive no repair and no second model call.

Every generated record remains outside production knowledge under
`scripts/research/articulation_out/review-pending/`, retains its support label,
and requires semantic and owner review.
