## False-friend check

The survey's false-friend list was recomputed before the merge. Classes were
selected only by their normalized documented-error value, never merely by a
shared registration name. Several retained names also occur on separate rows
with different documented errors, including `too_vague`, `add_across_unlike`,
`smaller_from_larger_columnwise`, and `smaller_from_larger_standard_alg`; those
rows remain separate and were not included in these unions.

## Articulation harness

The default model is `Qwen3-Coder-Next`. The drafting prompt now requires a
self-contained clause set with exactly one registration and one runnable rule,
no directives or `use_module`, and only `test_harness:` and
`churn_candidate:` module qualifications. The response contract distinguishes
source-determined `ARTICULATE`, explicitly marked `EDUCATED_GUESS`, and
`DECLINE` responses. An educated guess must name both its inferred commitment
and a falsification probe. Every generated candidate remains review-pending
regardless of gate passage.
