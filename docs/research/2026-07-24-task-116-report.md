## Big Red analysis

`scripts/bigred/strategy_algebra/analyze_strategy_algebra.py` is
self-contained and uses the Python standard library. It performs:

- finite labeled-transition bisimulation minimization for each observed
  signature;
- action-labeled canonicalization and structural-coincidence grouping;
- bounded shared-subtrace enumeration, with a default maximum length of six;
- rooted, action-preserving candidate homomorphism checks;
- bounded synchronous products for overlapping action domains, with a default
  limit of 100 reachable product states.

The script carries this rule in its output:

> Pedagogically distinct strategies are preserved even when minimized
> structures coincide. A coincidence is reported as a finding, never a merge.

Full mode compares \(69 \times 68 / 2 = 2{,}346\) pairs after 69
minimizations. Expected wall time is about 2 to 10 minutes on one Big Red CPU.
Shared-subtrace enumeration, homomorphism checks, and products depend on action
overlap, so the Slurm request allows 30 minutes, one CPU, and 4 GB.

The job follows the existing suite conventions. Account and email remain
fillable placeholders. Its lmod and conda block runs under `set +u`, then
restores `set -u`. The README requires creation of
`scripts/bigred/strategy_algebra/work/logs` before `sbatch`, because Slurm opens
log files before the job body begins. This task did not submit or run full
mode.
