# Task 120 report — substitution-license seed from the E343 vocabulary crosswalk

## What this is

`knowledge/crosswalk/vocabulary_licenses.pl` (new) transcribes the owner-
authored 79-entry, six-framework vocabulary crosswalk
(`/Users/tio/Documents/GitHub/Math-Methods/vocabulary_crosswalk_expanded.json`,
version `2.0`, dated `2026-03-31`) into flat Prolog facts. The crosswalk maps
Van de Walle, Indiana Academic Standards, CCSS, Illustrative Mathematics,
Five Practices (Smith & Stein), and CDM/Talk Moves (Chapin) terms for 79
shared concepts, with a `confusion_risk` (LOW/MEDIUM/HIGH) per entry that is
this task's design center: HIGH means the same word covers different things,
or different words cover the same thing, in ways the source found likely to
confuse students; LOW/MEDIUM means the frameworks name the concept
transparently enough to bound a substitution.

The `e343_alignment_graph.json`'s `terminology_crosswalk` block was checked
against the standalone file before transcription: parsed as JSON, the two are
identical (`==` true in Python across every key, including all 79 entries).
There is one source, not two to reconcile.

`knowledge/strategies/math/state_vocabulary.pl` was read as the house
precedent this module follows in spirit: attribute a term to the tradition
that authored it, cite it, and never collapse traditions into a single
"canonical" name. This module does the same for the crosswalk's six
frameworks, at the data layer only — it adds no display-selection logic like
`state_vocabulary.pl`'s `state_display_label/2`, because nothing outside this
module reads these facts yet.

## Schema

```prolog
vocabulary_license(ConceptId, Framework, Term, Risk, LicenseKind,
                    provenance(EntryIndex, SourceFile, SourceVersion)).
vocabulary_license_concept(ConceptId, EntryIndex, ConceptText).
vocabulary_license_note(ConceptId, Note).
vocabulary_license_source(SourceFile, Version, Date).
```

- `ConceptId` is `vl001`..`vl079`, one per source-array position (not a slug
  of the concept text — 79 auto-transcribed English descriptions do not
  reduce to unique short identifiers without inventing names the source
  never gave them; per Tio's ruling that his examples are illustrative and
  never enshrined literally, nothing here promotes an entry's incidental
  concept sentence into naming vocabulary).
- `Term` is the framework's cell **verbatim**, including its own "no term
  here" phrasing (e.g. `"Not addressed (content-neutral framework)"`) — nothing
  is normalized or paraphrased at the term level either.
- `Note` is the entry's authored note, verbatim, in its own fact — the
  pedagogical reasoning the note carries is not folded into a citation
  string or summarized.
- `LicenseKind` ∈ `{substitutable_in_context, disambiguation_required,
  not_addressed}`, decided per (entry, framework) cell, not per entry: an
  entry's overall risk sets the ceiling (`disambiguation_required` for HIGH,
  `substitutable_in_context` for LOW/MEDIUM), but any cell classified as
  carrying no term of its own is `not_addressed` regardless of the entry's
  risk.

### The not_addressed classification rule

A cell counts as `not_addressed` when its text contains one of a fixed set of
explicit source-literal disclaimer markers — `"not addressed"`, `"not
specified"`, `"not named"`, `"not distinguished"`, `"not referenced"`, `"not
described"`, `"not directly"`, `"not explicitly"`, `"not used"`, `"not a
curriculum component"`, `"not systematically aligned"`, `"implied via"` — or
matches one oddball descriptive negative that names no term of the
framework's own without using the word "not" as its lead
(`"references ccss codes, not indiana codes"`). This rule is mechanical and
source-literal, not a semantic judgment about term quality, and it is
implemented once, in `scripts/checks/vocabulary_licenses.py`, which
re-derives every entry's expected classification from the untouched source
JSON on every run rather than trusting the `.pl` file once written.

**The honest boundary**: descriptive cells that name no proper term but also
use none of those markers are counted as carrying a term. For example, entry
24 (revoicing)'s Five Practices cell reads `"Part of Monitoring and
Connecting"` — the entry's own note says outright that "Five Practices
embeds discourse within its practices without naming specific moves" — yet
this cell is classified `substitutable_in_context`/`disambiguation_required`
(whichever the entry's risk sets), not `not_addressed`, because it contains
no marker. The same holds for `"Embedded in curriculum design (tends toward
guided invention)"` (entry 39, IM), `"Monitoring (tracking, assessing,
advancing)"` (entry 11, Five Practices — this one plausibly is a real,
named term), and about two dozen cells like them. Reclassifying these would
require judging whether embedded, descriptive prose "really" names a term —
a semantic call this task does not make. The rule is disclosed here exactly
so that judgment is visible and revisable, not silently baked in.

## Census + lexicon join (candidates only — nothing applied)

Per the brief, this seed is joined against three read-only surfaces by
conservative exact/stem match: the automata label census
(`python3 scripts/research/strategy_label_census.py`, both the default JSON
form and `--format proposal-csv`), `scripts/research/mobius_band_lexicons.json`,
and `scripts/research/brandomian_lexicons.json`. The census currently reports
638 unique action labels and 51 unique state labels across 172 registered
signatures (matching the task-116 count); the two lexicon files together
name 80 unique anchor phrases (82 raw phrase entries, 2 duplicated across
files: `"measure"`, `"cannot"`).

**Match rule** (documented so the counts are reproducible, not just
asserted): tokenize the crosswalk `Term` into lowercase words of length ≥ 4,
drop a fixed ~70-word stoplist of generic pedagogical vocabulary
("student", "practice", "standard", "not", "using", ...), and stem what
remains with the same `stem_token` the wave's own label census already uses
(`scripts/research/strategy_label_census.py:stem_token`). Do the same for
each automaton label (split on `_`) and each lexicon anchor phrase (split on
whitespace). A label or anchor **touches** a term only when the label's/
anchor's *entire* stemmed token set is contained in the term's token set —
containment, not overlap. An early looser rule (any shared token) was tried
first and rejected: it touched 451/638 action labels by flooding on generic
one-word coincidences (`"take"` matching `take_absolute_distances`,
`"construct"` matching `construct_ratio_diagram` for a "construct viable
arguments" cell that means something else entirely). Containment is the
conservative choice the brief asks for; the price is the honest one below.

### What the touches look like, and what they are worth

Some are the finding the wave already named: `"Partition division (sharing
division)"` and `"Partitioning / Sharing"` (VdW/Indiana, entry 5) both touch
`q_partition`/`q_partition_set` and the action label `partitions` — the same
"partition doing double duty" the source's own note calls out for fraction
vocabulary. `"Compare"` (entry 4) touches the action label `compare`
directly. `"3.NS.3: Model a non-unit fraction on a number line..."` (entry
32) touches `q_mark_off_lengths` and the anchors `number line`, `length`,
`mark`, `is equal to` together — a plausible real cluster around the
measurement-construct automata.

Others are homograph collisions that the rule cannot distinguish from real
overlap, and are exactly why the brief calls these candidates rather than
licenses: `"Know from memory"` (entry 7, CCSS, on automaticity) touches the
disfluency anchors `"i don't know"` / `"i do not know"` on the single shared
word "know" — a fluency term and a student hedge marker, not the same
phenomenon. `"Monitoring (tracking + assessing student thinking)"` (entry
20) and `"Think-Pair-Share / Partner discussion"` (entry 66) touch `"i don't
think"` / `"i do not think"` the same way, on "think"/"thinking". `"PS.1:
Make sense of problems..."` touches the anchor `make` on a single generic
verb. These are reported, not filtered, because filtering them by hand would
smuggle back the semantic judgment the mechanical rule is supposed to avoid
making silently.

**Honest reading**: this 79-entry pedagogical-vocabulary seed and the
638-action/51-state automaton label vocabulary live at different levels of
description — problem-type and discourse-move naming versus procedural
micro-step naming — the same abstraction-layer mismatch the task-116/119
wave findings already diagnosed for the automaton labels among themselves.
Under a genuinely conservative match, this seed reaches under a quarter of
either label surface and about a quarter of the two lexicon files, and a
meaningful share of what it does reach is coincidental rather than
substantive. That is the honest count the brief asked for, not a shortfall
in the matching effort.

## The quarantine boundary

Every fact in `vocabulary_licenses.pl` is inert data. Nothing in
`hermes_worker.pl`, `hermes/dispatch_spec.pl`, the reader
(`hermes/math_claim_language.pl`), any recognizer, or any automaton imports
this module — confirmed by grep (only `scripts/checks/vocabulary_licenses.py`
and this report reference it). Wiring a `substitutable_in_context` entry into
an actual term substitution anywhere — the reader's grammar, a recognizer's
label matching, a misconception mapping, the crosswalk's own `edge/5` family
rows — is a formal-core change out of scope here and needs its own reviewed
slice. A `disambiguation_required` entry must never become a synonym
expansion in any consumer, present or future; this module records the
obligation, it does not discharge it.

## New check: `scripts/checks/vocabulary_licenses.py`

Not wired into `scripts/checks/run_all.sh` in this slice (the brief's file
fence excludes that file). It:

1. Re-reads the source JSON fresh and re-derives, independently of the `.pl`
   file, what every one of the 474 `(entry, framework)` cells should say —
   concept text, note text, term text, risk atom, and `LicenseKind` — using
   the documented classification rule above, then diffs that expectation
   against what the parsed Prolog facts actually contain. A dropped fact, a
   copied-wrong risk atom, or a paraphrased note fails the round trip.
2. Confirms exactly one `vocabulary_license_source/3` fact, agreeing with
   the source's own file name / version / date.
3. Confirms every risk atom is one of `{low, medium, high}` and matches the
   source's `confusion_risk` for its entry; confirms every `LicenseKind` is
   one of the three documented atoms; confirms every license fact's
   `provenance/3` names the correct entry index and source file/version (a
   malformed or missing provenance fails to parse at all, which the check
   treats as a hard failure rather than silently skipping the line).
4. Runs the strict SWI-Prolog load exactly as specified:
   `swipl -q --on-warning=status --on-error=status -l paths.pl -g
   "user:ensure_loaded(crosswalk(vocabulary_licenses)), halt."`
5. Re-invokes the two documented, read-only census CLI forms
   (`--format json` default and `--format proposal-csv`) as a live
   regression that they still run clean, then imports
   `strategy_label_census.py` directly (as `scripts/checks/
   mobius_band_readers.py` already does for its own driver) to compute the
   join in-process.
6. Computes the join twice in the same run and requires byte-identical JSON
   serialization (the two-run determinism the brief's Verification section
   asks for); a second, separate CLI invocation of the whole check was also
   diffed byte-for-byte against a first run (see Verification).

A negative-control test (not part of the shipped check; run once during this
task and discarded) confirmed the check actually catches drift: corrupting
a scratch copy of the `.pl` file by dropping one license fact and changing
one risk atom from `high` to `medium` produced exactly those two diagnostics
and a nonzero exit, against the untouched real files.
