# Task 96 report — input contracts for the automata registry

## Implemented surfaces

- Added `knowledge/strategies/automaton_input_contracts.pl`. Each fact names an operation and kind, a typed JSON input template, a concrete JSON example, and `verified(strategy_trace_ok)`.
- `hermes/mcp/server.py` reads those facts directly, without a new Prolog or worker dependency at schema-build time. `strategy_trace` now exposes the template, worked example, and verification method in each contracted strategy description. Its could-not-run response includes the same data for close contracted alternatives.
- Added `scripts/bigred/suite/run_modeling.sh` and `run_modeling.py`. The runner executes only contracted signatures, writes one JSON result table per signature under `.bigred-output/modeling/`, derives a bounded input neighborhood from the input shape, and reports the uncontracted remainder. `--limit N` limits signatures for smoke runs.
