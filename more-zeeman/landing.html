What happens when a way of knowing stops working?

A computational agent begins with counting-by-ones and a small inventory of action automata. When a problem cannot be solved within resource limits, it enters crisis — and either consults a teacher or composes a different strategy from primitives already in its registry. These views show the same process from different angles: the geometry of the snap, the formal dynamics of the Prolog, the step-by-step narrative, and the representation surfaces that make each mathematical move visible.

What this is

Hermes is a local console for teachers of elementary mathematics. It runs on your machine and reads a knowledge base that encodes children's arithmetic strategies as runnable machines, misconceptions documented in research literature, content standards, and lesson context from the Illustrative Mathematics curriculum. When you ask it something, the answer comes from that knowledge base, with its sources attached.

Console

Type a computation and the calculator works it the way a child using a named strategy would, drawing each action on a number line. The encyclopedia holds strategies, documented misconceptions, standards, grounding metaphors, and the places where each metaphor stops working. The lesson lab gathers what the knowledge base holds about a particular IM lesson.

Discussions

Paste a classroom transcript and the console checks the mathematical claims it can recognize, deterministically: each claim comes back holds, refuted, or not covered, with the worked check attached. The fuller two-pass report asks a language model first, so it needs a key.

Visualizers

Number line, fraction bars, area model, base-ten blocks, set & grouping, and balance scale. The same drawing grammar the calculator uses, available directly.

Monitoring charts

For an IM lesson: its standards, the strategies to anticipate, and the misconceptions research has documented near its content, arranged the way a monitoring sheet arranges them.

Pick your door

This is the public face of a workbench for how children learn arithmetic — counting by ones until it stops working, and the moment after, where a new strategy is built. Everything behind these doors is hostable and needs no setup. The one tool that reads student work runs only on your own machine, and it says so.

These registers sit side by side. Choose by what you need to do; none is the preferred route.

How they connect

Geometry ← Prolog tension → Narrative

The More Machine shows the topology of strategy transitions. Catastrophe theory describes how smooth changes in control parameters produce discontinuous jumps in behavior. You feel this directly — the disc resists, then gives way.

The Bridge maps the Prolog system's internal state onto that topology. The meta-interpreter's inference count becomes tension. Crisis classification becomes the control parameter. Strategy acquisition becomes the snap. The geometry is no longer abstract — it is driven by an actual computation trying to learn arithmetic.

The ORR Explorer shows the same process as a story. Where the Bridge gives you dynamics, the Explorer gives you reasons: why the system entered crisis, what the teacher provided, how the new strategy was synthesized, whether the retry succeeded. The three views are the same phenomenon — the geometry, the dynamics, and the account.

How the three vocabularies stay in step

The project keeps three vocabularies in step: the philosophy that names a commitment, the modal calculus that codes it, and the Prolog that runs it until it breaks. This section follows one documented student error through all three, so a reader can check that the same error is being described each time and is not quietly changing under the translation.

tasksolve  6 + X = 4
studentimpossible ("you cannot add to 6 and end up with less")
correct-2 (6 + (-2) = 4)
1 · Philosophy — the commitment

"A sum must be larger than its parts."

The student is committed to a categorical claim about addition: putting numbers together can only grow the total. In Brandom's terms this is a material-inferential commitment the student is not entitled to. It holds for counting collections and fails the moment an addend is negative, so the move from a true case to a categorical rule is the unlicensed step.

The corpus row records the grounding the commitment leans on with a Source-Path-Goal schema: addition read as travel along a path that only ever moves forward.

2 · Logic · PML — the coding

A subjective, categorical commitment held without entitlement

PML is a material-inference calculus, not a Kripke modal logic. It tags the commitment with a mode, a polarity, and an entitlement marker. The wiring records this misconception as:

s( comp_nec( unlicensed( addition_must_make_larger ) ) )
  • s(·) — Subjective validity: read from the student's own position, not yet checked against the object or the norm.
  • comp_nec(·) — Compressive Necessity (↓): a categorical "must", a fixation or crystallization rather than an opening.
  • unlicensed(·) — the commitment is asserted without entitlement: the categorical rule is not something the student has earned the right to.
Source: misconceptions/pml_wire.pl — misconception_pml(db_row(193), …), one of 820 wired facts; legend in pml/pml_operators.pl.
follow it yourself
swipl -l paths.pl -g \
  "use_module(misconceptions(pml_wire)), \
   misconception_pml(db_row(193),Op), \
   format('~q~n',[Op]), halt"
% =>  s(comp_nec(unlicensed(addition_must_make_larger)))
↓ run as
3 · Math · Prolog — the break, made runnable

The rule returns the wrong token exactly at the boundary

The misconception is encoded as a rule that fires on the task. On the missing-addend case it returns impossible, while the value recorded alongside it is -2. The categorical "must make larger" breaks precisely where the addend goes negative, and the Prolog does not smooth that over. It returns the wrong token at the boundary, which is where the misconception becomes detectable rather than hidden.

r193_addition_must_make_larger(missing_addend(6,4), impossible). arith_misconception(db_row(193), whole_number, addition_must_make_larger, …, -2).
Source: misconceptions/misconceptions_whole_number_batch_5.pl — the rule predicate and the arith_misconception/6 row carrying the correct value.

What this section is, and is not

  • One worked example, not a live engine. The three-layer trace is hand-chosen and authored. The facts in each panel are real and queryable, but the translation from philosophy to logic to Prolog is not discovered end-to-end by a program; a person assembled it from the tracked predicates.
  • The PML coding is a proposal to test. The discourse-marker mapping behind misconception_pml/2 is a coding scheme under test, not a validated instrument. Every one of the 820 wired rows currently carries the same s(comp_nec(unlicensed(_))) shape, so the coding marks a family resemblance, not a measured distinction. Read it as a hypothesis the project is checking.
  • The break is the point. The boundary where the rule returns the wrong token is not a defect to hide. It is the place the formalization names where a categorical commitment stops holding.
  • Partial. The fuller interactive walkthrough planned for the paper page is not built here; its framing lives in a source this build does not draw on. This static section closes the narrower goal of letting a reader trust the chain on a single, reproducible example.

What this is

This is a research formalization, not a finished product. A Prolog system models how a learner acquires arithmetic strategies through crisis — beginning with counting-by-ones, consulting a teacher when stuck, composing strategies from action automata already in its registry. The model covers whole-number arithmetic through the full ORR cycle (Observe, React, Reorganize). For fractions, the registry now includes the area model for fraction multiplication — fraction:area_model_part_of_part — and a paired productive/deformation rule that captures Van de Walle's "rules without reason" distinction: cross_multiplication_rule_from_pattern carries an area-model justification in its trace; cross_multiplication_rule_without_ground produces the same numeric answer with no inferential ground.

What still does not work is the negotiation: the meta-interpreter does not yet route fraction outcomes through the ORR cycle as recollection-compatible values, and metamorphic accommodation (Norton/Steffe — strategies synthesised by recursive composition of others) is not implemented. So the system can act with fractions, but it cannot yet undergo the crisis of acquiring them. The formalization is a tool for noticing what it can and cannot do, with equal specificity.
