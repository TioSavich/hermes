Previously: Counting got too expensive and something had to give. You felt the snap.

After the crisis, the child discovers shortcuts. Not because someone told them to — because counting all 93 marks was too much.

First shortcut: instead of counting all from the beginning, start from the bigger number. 38 + 55? Start at 55. Count on: 56, 57, 58... That's 38 steps instead of 93.

Better shortcut: 38 is close to 40. Move 2 from the 55 to make 40 + 53. Now you're working with a round number — rearranging to make a base. That costs about 5 steps.

Even better: what about 6 x 14? Break 14 into chunks: 6x10 + 6x4. That's chunking: splitting a hard problem into easy ones (Steffe and Olive, Children's Fractional Knowledge).

Each shortcut is cheaper than the last. But notice: each one is also more clever. Counting-all is mechanical. RMB requires you to see that 38 is near 40 — that's a judgment, not a procedure.

The CGI research program (Carpenter, Fennema, Franke, Levi, & Empson, 1999) documents a developmental sequence of strategies for whole-number arithmetic. Each strategy is more efficient but also more abstract:

Counting-all = Direct Modeling. The child physically or mentally represents every unit. Cost: O(A + B).

Counting-on = Counting strategy. The child holds the larger addend as a cardinal value and counts forward. Cost: O(min(A, B)).

COBO = bases-then-ones (a Derived Fact strategy). Decompose into tens and ones, add each place separately: 30+50=80, 8+5=13, 80+13=93. Cost: O(1) for each decomposition step.

RMB = rearrange to make a base-10 number. Compensate: 38+55 = 40+53 = 93. Requires recognizing proximity to a round number. Cost: O(1) for the compensation.

Note: COBO is "bases-then-ones" — not "counting on by ones." The child works with tens and ones as separate units, combining them at the end. This is a derived fact strategy, not a counting strategy. Hackenberg (via Steffe) distinguishes these carefully.
Biddlecomb, B. D. (1994). Theory-based development of computer microworlds. Journal of Research in Childhood Education, 8(2), 87-98. Steffe & Olive (1996). Symbolizing as a constructive activity in a computer microworld. Journal of Educational Computing Research, 14(2), 113-138. These researchers connected Steffe's constructivist schemes directly to software architecture — the line of work this system continues.

The key observation: each strategy invokes the one below it as a subroutine. COBO decomposes into place-value components, then uses counting (or direct knowledge) to add the ones. RMB compensates, then adds the adjusted values — which may itself use COBO. Strategies nest inside strategies.

Each strategy in the Prolog formalization is a finite-state machine. A genuine synthesis engine would construct these machines from ENS primitives: partition, disembed (pull a part back out of the whole), and iterate (Steffe and Olive, Children's Fractional Knowledge). The synthesis attempt was archived in Phase 1 (it wrapped oracle calls rather than synthesizing); fsm_synthesis_engine.pl is now only a utility shim. Genuine FSM synthesis from primitives remains an open research direction.

The distinction matters philosophically: an oracle-backed strategy is a phone number the learner memorized. A genuinely synthesized strategy is a method the learner built. The system currently cannot tell the difference from the inside — both look like working FSMs. Only the provenance differs.

Negarestani, R. (2018). Intelligence and Spirit. Urbanomic/Sequence Press. Negarestani distinguishes two kinds of navigation: (1) physical/causal navigation (the automaton's protosemantic encounter with items in space) and (2) cognitive navigation of logical space ("navigating the coherentist network or logical space of truth-candidates"). The meta-interpreter's strategy selection is the beginning of cognitive navigation: choosing among strategies based on their cost structure, not just executing the first one found.
Negarestani on the Chomsky hierarchy: a strategy that calls a sub-strategy is a pushdown automaton — it needs a stack to track where it came from. The nesting of strategies is not just a convenience; it is a move from regular to context-free. "True intelligence transcends merely navigating physical space via causal inputs; it involves a cognitive navigation of the 'coherentist network or logical space.'"

The fractal architecture: each circular automaton (the counting PDA, the COBO machine, the RMB compensator) can invoke any machine below it as a subroutine. COBO calls counting for its ones phase. RMB calls COBO after compensating. This is strategies nesting inside strategies — the same structure the fractal visualization depicts as concentric rings.

But each machine is still flat. It has a fixed number of states, a fixed transition table, a stack of fixed depth. What happens when the nesting goes deeper — when you need a strategy that dynamically composes strategies that themselves compose strategies? That requires something the current FSM architecture cannot provide.
